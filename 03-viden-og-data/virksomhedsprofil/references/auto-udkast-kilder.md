# Auto-udkast: offentlige kilder og troværdighedsregler

Metoden bag trin 2 i arbejdsgangen: hent alt hvad der kan hentes, FØR interviewet, så ejeren retter frem for at diktere. Læs denne fil før hvert auto-udkast.

## Grundregler

1. **Budget: maks. 2 opslag pr. kilde.** Fejler et opslag to gange, notér "[MANGLER - kilde utilgængelig ÅÅÅÅ-MM-DD]" og gå videre. Brug aldrig tid på at fejlfinde netværk eller lede efter alternative spejle.
2. **Ingen web-adgang = intet auto-udkast.** Sig det ærligt og gå til interview. Skriv ALDRIG "hentet fra ..." om noget du har fra hukommelsen - din træningsviden om en konkret dansk virksomhed er pr. definition forældet og ukontrollerbar.
3. **Hvert hentet faktum bærer kilde + dato:** `(kilde: datacvr.virk.dk, 2026-07-07)`. Uden kilde kan ejeren ikke efterprøve, og brain-tjek kan ikke vurdere forældelse.
4. **Kilderne er udgangspunkter, ikke sandheder.** Alt fra nettet vises til ejeren i diffen. Hjemmesider overdriver; registre halter bagefter.

## Kildeliste i prioriteret rækkefølge

### 1. datacvr.virk.dk (Det Centrale Virksomhedsregister)

Autoritativ for stamdata. Slå op på CVR-nummer eller firmanavn.

| CVR-felt | Profilfelt | Bemærkning |
|---|---|---|
| Juridisk navn + binavne | Stamdata: juridisk navn | Brug det juridiske navn, notér binavne |
| CVR-nummer | Stamdata: CVR-nummer | |
| Virksomhedsform | Stamdata: virksomhedsform | ApS, A/S, enkeltmandsvirksomhed osv. |
| Branchekode | Stamdata: branchekode | NACE-kode + tekst; spørg ejeren om den stadig passer |
| Antal ansatte | Stamdata: antal ansatte | Ofte interval (fx "10-19") og forsinket - markér "pr. [CVR's opgørelsesperiode]" og bed ejeren præcisere |
| Adresse | Stamdata: adresse | |
| Startdato | Stamdata: stiftelsesår | |
| Registreret ledelse/ejere | Stamdata: ejer/direktør | Kun det offentligt registrerede |
| Status | (kontrol) | Er status ikke "Normal/Aktiv": stop og spørg ejeren hvad der foregår |

Troværdighed: høj for juridiske fakta, lav for aktualitet (ansatte, branche kan halte 1-2 år).

### 2. Virksomhedens egen hjemmeside

Primærkilde til ydelser, segmenter og selvforståelse. Hent i denne rækkefølge, og stop når du har nok:

1. **Forsiden** - positionering og hovedydelser
2. **Om os** - historie, team-størrelse, værdier
3. **Ydelser/produkter/priser** - til Produkter og ydelser-sektionen; offentlige listepriser noteres med "pr. [dato]"
4. **Kontakt** - adresse-krydstjek mod CVR; afviger de, flag det som åbent spørgsmål

Troværdighedsregler for hjemmesideindhold:

- **Ydelseslisten:** troværdig som bruttoliste, men skelner ikke mellem det de sælger meget af og det der bare står der. Omsætningsfordeling SKAL komme fra ejeren.
- **Marketingsprog oversættes:** "totalleverandør af digitale løsninger" er ikke et faktum - udkast en konkret formulering og markér den [ANTAGET: tolket fra hjemmesidens ydelsesliste].
- **Kundelogoer/referencer:** notér som "[ANTAGET: hjemmesiden viser referencer fra X, Y]" - logoer beviser ikke aktive kundeforhold.
- **Alt dateres:** hjemmesider har ofte forældede sider liggende. Ligner en side noget fra en tidligere æra (gamle årstal, fratrådte medarbejdere), flag det frem for at citere det.

### 3. Supplerende kilder (kun hvis hullerne kræver det)

- **LinkedIn-virksomhedsside:** medarbejderantal (tæl profiler tilknyttet virksomheden - notér som indikation, ikke facit), aktuelle aktiviteter. Aldrig persondata om enkelte medarbejdere ind i profilen.
- **Google Maps/anmeldelser:** åbningstider, kundetype-indikationer. Citér aldrig enkeltanmeldelser i profilen.
- **Konkurrent-research:** søg "[branche] [by]" for at udkaste 2-3 konkurrent-kandidater, alle markeret [ANTAGET: fundet ved søgning - bekræft at I reelt møder dem]. Ejeren kender sine konkurrenter bedre end søgeresultater gør.

## Hvad auto-udkastet ALDRIG henter

- Regnskabstal fra CVR/offentliggjorte årsrapporter - medmindre ejeren eksplicit beder om det. Profilen deles bredt, og tallene inviterer til fejltolkning uden kontekst.
- Persondata om medarbejdere (navne fra "mød teamet"-sider tages KUN med hvis ejeren godkender det i diffen).
- Noget som helst fra betalingsmurede registre eller kilder der kræver login.

## Felt-for-felt: forventet dækning

| Profilsektion | Auto-udkast kan levere | Kræver ejeren |
|---|---|---|
| Stamdata | 80-100 % (CVR) | Præcisering af ansatte |
| Forretningsmodel | 10-30 % (kun indikationer) | Omsætningsfordeling, prismodel, sæson |
| Kunder og segmenter | 20-40 % (hjemmesidens målgruppesprog) | Faktiske segmenter + fravalg |
| Produkter og ydelser | 60-80 % (hjemmesidens liste) | Prioritering, reelle priser |
| Konkurrenter | 20-30 % (søge-kandidater) | Navne + differentiering |
| Systemlandskab | 0-10 % | Stort set alt |
| Nøgleprocesser | 0 % | Alt |

Brug tabellen til at planlægge interviewet: systemlandskab og nøgleprocesser SKAL interviewes, stamdata skal typisk kun bekræftes.
